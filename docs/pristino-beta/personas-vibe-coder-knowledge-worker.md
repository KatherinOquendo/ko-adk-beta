# Personas: Vibe Coder y AI-native Knowledge Worker

## Por que Personas

Beta separates universal harness logic from active profiles. [DOC] This avoids
hardcoding one brand, one style or one type of deliverable into the core.

## Vibe Coder

The vibe coder builds software in tight build-run-observe loops. [DOC] The goal
is working software with proportional ceremony, not casual hacking.

Default behavior:

- Quick depth first. [DOC]
- Working code before long prose. [DOC]
- Tests, security and evidence never bend for speed. [DOC]
- Estimates use decomposition and explicit assumptions. [DOC]

Good tasks:

- Build a mini app.
- Prototype a workflow.
- Repair a small feature.
- Scaffold an internal tool.
- Turn a rough need into a runnable app slice.

## AI-native Knowledge Worker

The AI-native knowledge worker delegates research, synthesis, decision support,
documents and operations. [DOC] The priority is defensible conclusions.

Default behavior:

- Balanced depth. [DOC]
- Every non-trivial claim tagged and cited. [DOC]
- Provenance preserved. [DOC]
- Assumptions surfaced, not laundered into facts. [DOC]

Good tasks:

- Research a domain.
- Create an operating brief.
- Compare tools or vendors.
- Produce a sourced decision memo.
- Turn notes into a playbook or FAQ.

## Difference from Alfa

Alfa is broad and public. [CONFIG] It works well as a full local development kit
for current site/app/Hostinger workflows.

Beta is compact and profile-driven. [DOC] It aims to make the same operating
discipline available to more contexts without loading the whole Alfa surface.

## Selection Rule

| If the task asks for... | Choose |
|---|---|
| runnable code, mini app, prototype, UI, integration | Vibe coder |
| research, synthesis, brief, source-backed decision | Knowledge worker |
| MetodologIA public/commercial deliverable | MetodologIA profile |
| ambiguous brand/profile | Halt and ask |

## Failure Modes

- Using vibe-coder to produce unsourced strategic claims. [INFERENCIA]
- Using knowledge-worker to over-document a tiny code fix. [INFERENCIA]
- Mixing profiles in one output. [CONFIG]
- Calling Beta public before gates pass. [CONFIG]
