# Runtime · Tool-Access Matrix

How each supported runtime reaches the harness's single MCP server, **google-workspace**
(`uvx workspace-mcp`, env-var auth, services: gmail · drive · calendar · docs · sheets · slides).
Config templates are generated by `scripts/generate-mcp-configs.py --apply` into
`references/mcp/` and validated by `scripts/validate-mcp-config.py`. [DOC]

| Runtime | Adapter / rules | MCP config template | Notes |
|---|---|---|---|
| **Claude Code** | `CLAUDE.md` | `.mcp.json` (project scope, canonical) | Source of truth for all templates. [CONFIG] |
| **Claude Desktop** | `CLAUDE.md` | `references/mcp/claude_desktop_config.json.example` | `mcpServers` block. |
| **OpenAI Codex** | `AGENTS.md` | `references/mcp/codex.config.toml.example` | `codex mcp add google-workspace -- uvx workspace-mcp`. |
| **Gemini CLI** | `GEMINI.md` | `references/mcp/gemini.settings.json.example` | `mcpServers` block. |
| **Antigravity** | `.agent/rules/GEMINI.md` | `references/mcp/antigravity.mcp_config.json.example` | `mcpServers` block. |
| **Cursor** | `CLAUDE.md` | `references/mcp/cursor.mcp.json.example` | `mcpServers` block. |
| **Windsurf** | `CLAUDE.md` | `references/mcp/windsurf.mcp_config.json.example` | `mcpServers` block. |
| **VS Code** | `CLAUDE.md` | `references/mcp/vscode.mcp.json.example` | `servers` block (VS Code schema). |

**Auth** `[CONFIG]`: env-var only — no literal tokens in any template (enforced by the
validator's secret scan). **Anti-scope** `[DOC]`: templates are examples; the live config
is each runtime's own file. Regenerate after any `.mcp.json` change; never hand-edit the
`*.example` files.
