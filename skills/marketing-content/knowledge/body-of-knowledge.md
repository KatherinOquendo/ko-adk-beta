# Body of Knowledge — marketing-content

Domain knowledge for routing and executing the eight marketing-content
playbooks. Scope is a **router**: pick one topic, run one playbook. [DOC]

## 1. The eight topics and their canonical structures

| Topic | Canonical structure | Hard rule |
|-------|---------------------|-----------|
| case-study-writing | Headline → Snapshot → Problem → Solution → Result → CTA | Hero metric is a business outcome with before/after; quote verbatim+approved |
| content-calendar | Pillars → clusters → dated slots → repurposing chains | 3–5 pillars; ~20% buffer; owner+metric per slot |
| copywriting-frameworks | AIDA / PAS / FAB / 4Ps / BAB (pick one) | One framework per asset; single repeated CTA |
| event-marketing | Webinar / conference / CFP / follow-up sub-domains | One CTA per asset; segment follow-up by behavior |
| podcast-prep | Arc → segments → tiered questions → run-of-show → show notes | Fact-check flag on every borrowed stat |
| press-release | Inverted pyramid: headline → lede → body → boilerplate → `###` | Lede ≤35 words; no unsourced superlatives |
| video-script | Hook 0–3s → body beats → CTA 3–5s, two-column VISUAL\|VO | ≈2.5 words/sec budget; open captions `[CC]` |
| whitepaper-creation | Thesis → methodology → findings → recommendations → citations | Single thesis in first 150 words; citations resolve |

## 2. Routing decision rules

- **One topic per run.** If two plausibly fit, ASK; never run two playbooks. [INFERENCIA]
- Keyword presence is not intent — confirm the intended *artifact*. A "case
  study" with no before-number is a testimonial, not a case study. [INFERENCIA]
- Awareness drives the copywriting framework: unaware→AIDA, pain-aware→PAS,
  comparison→FAB, proof-rich→4Ps, outcome→BAB. [DOC]
- Anti-scope (redirect, do not absorb): brand voice/tone, internal comms, sales
  decks, SEO keyword research, paid-media bidding, visual/layout design. [DOC]

## 3. Evidence taxonomy (Alfa set)

| Tag | Meaning |
|-----|---------|
| `[EXPLICIT]` | Stated directly by the user or a primary source |
| `[DOC]` | Drawn from a reference playbook, config, or doc |
| `[INFERENCIA]` | Reasoned conclusion, not directly stated |
| `[SUPUESTO]` | Assumption requiring verification before publish |

Every claim carries exactly one tag. Quantitative claims cite a source or are
marked `[SUPUESTO]` with a verification step. [DOC]

## 4. Governance standards (Constitution v6.0.0)

- **Think First (XIII):** resolve audience/CTA/structure before drafting.
- **Simple First (XIV):** cadence and scope from real capacity, not ambition.
- **Script-first / deterministic checks** run before subjective review.
- No invented prices, ROI, or projections — placeholders only.
- Single-brand per output; no PII; superlatives need an attributable source.
- Never green-as-success: report pending placeholders and open `[SUPUESTO]`. [DOC]

## 5. Cross-topic asset flow

A single source asset fans out: whitepaper/case-study (source) → content-calendar
slots → copywriting (slot copy) → video-script / podcast-prep (repurposed) →
press-release / event-marketing (timely amplification). The router still loads
only the one topic requested. [INFERENCIA]
