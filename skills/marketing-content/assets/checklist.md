# marketing-content — Routing & Delivery Checklist

A fast pre-return checklist for any marketing-content run. Pair it with
`assets/quality-rubric.json` for scored review.

## Route
- [ ] Resolved exactly one `topic` from the 8-value enum.
- [ ] If two topics fit, asked one clarifying question before routing.
- [ ] Loaded exactly one playbook from `routes.json`; no siblings loaded.
- [ ] Confirmed a single brand (MetodologIA).

## Discover
- [ ] Gathered the resolved playbook's required inputs.
- [ ] Flagged every missing required input instead of assuming.

## Execute
- [ ] Drafted to the playbook's canonical structure (no cross-playbook mixing).
- [ ] Led with benefit over feature; single CTA where applicable.
- [ ] Tagged every claim ([EXPLICIT]/[DOC]/[INFERENCIA]/[SUPUESTO]) inline.

## Validate (governance — applies to all topics)
- [ ] No invented prices, ROI, or projections.
- [ ] No unsourced superlatives ("best", "leading", "revolutionary").
- [ ] Single-brand; no client PII.
- [ ] Unapproved quotes held as `[NEEDS APPROVAL]`.
- [ ] Every `[SUPUESTO]` resolved or explicitly disclosed.
- [ ] Verdict is evidence-backed; never green-as-success.
