# Podcast Prep
> "Method over hacks."
## TL;DR
Episode planning, guest research, question frameworks, show notes — a repeatable prep pass that turns a guest + topic into a recording-ready brief. [DOC]

## Procedure
### Step 1: Discover
- Capture episode goal, target audience, and the one takeaway a listener should keep. [DOC]
- Pull guest dossier: bio, 3-5 prior interviews/talks, recent work, contrarian or signature takes. [DOC]
- Note hard constraints: target runtime, recording date, format (solo/interview/panel), distribution channels. [DOC]
### Step 2: Analyze
- Pick an arc (chronological, problem→solution, myth-busting, or thesis-and-tension) per Constitution XIII/XIV. [INFERENCIA]
- Map 4-6 segments to runtime: ~1.5-2 spoken min per 250 words; a 40-min episode ≈ 6-7k words. [INFERENCIA]
- Draft questions in tiers: opener (warm, concrete), core (3-4 that advance the thesis), and 2 spares for dead air. [DOC]
### Step 3: Execute
- Write the run-of-show: cold open, intro, segments with time boxes, ad/break markers, outro + CTA. [DOC]
- Prepare host notes: pronunciation of guest name, links to cite, stats to verify, follow-up prompts under each question. [DOC]
- Tag every borrowed fact or stat for fact-check before publish; never read an unverified number on air. [SUPUESTO]
### Step 4: Validate
- Dry-run the opener and one core question aloud; confirm the arc still lands and runtime fits the box. [INFERENCIA]
- Confirm show notes: episode summary, timestamps, guest links, CTA, and any disclosures. [DOC]

## Quality Criteria
- [ ] Evidence tags applied to every borrowed fact/stat
- [ ] Arc and segment plan map to target runtime (±10%)
- [ ] Questions tier opener/core/spare; open-ended, not yes/no
- [ ] Guest name pronunciation and links verified
- [ ] Show notes draft includes summary, timestamps, CTA, disclosures
- [ ] Constitution-compliant, actionable output

## Usage

Example invocations:

- "/podcast-prep" — Run the full podcast prep workflow
- "podcast prep on this project" — Apply to current context
- "podcast prep, 30-min solo episode on X" — Constrain format and runtime up front

## Assumptions & Limits

- Assumes access to project artifacts (code, docs, configs) and a named guest/topic. [SUPUESTO]
- Requires English-language output unless otherwise specified. [DOC]
- Produces a prep brief and show-notes draft — NOT audio editing, booking, publishing, or transcription. [DOC]
- Stats/quotes are surfaced for verification, not asserted as fact; final fact-check is the host's. [SUPUESTO]
- Does not replace domain expert judgment for final editorial decisions. [DOC]

## Edge Cases

| Scenario | Handling |
|----------|----------|
| Empty or minimal input | Request clarification before proceeding |
| Conflicting requirements | Flag conflicts explicitly, propose resolution |
| Out-of-scope request | Redirect to appropriate skill or escalate |
| No guest dossier available | Proceed with topic-only arc; flag thin guest context as a gap |
| Runtime ≠ question volume | Re-tier questions or cut a segment; never overrun the box |
| Sensitive/contested topic | Prep neutral framing + disclosure; flag for host review before record |
| Unverifiable stat or quote | Mark `[SUPUESTO]`, exclude from script until confirmed |
