# Content Calendar

> "Method over hacks. Evidence over assumption."

## TL;DR

Build an editorial calendar from content pillars and topic clusters, set a sustainable cadence, and schedule across channels with repurposing built in. Outputs follow MetodologIA brand standards and carry `[EXPLICIT]` evidence tags. Plan to a stated horizon (default one quarter); each slot names pillar, channel, format, owner, publish date, and a measurable goal. [EXPLICIT]

## Procedure

### Step 1: Discover
- Capture audience, channels, brand pillars, cadence target, and quarter goal from user input. [EXPLICIT]
- Read existing posts, briefs, analytics, and configs to find current state, gaps, and best-performing topics. [EXPLICIT]
- Inventory reusable assets (whitepapers, case studies, talks) — each seeds a cluster, not a single post. [EXPLICIT]

### Step 2: Analyze
- Define 3-5 content pillars; map every topic to exactly one pillar to keep focus measurable. [EXPLICIT]
- Cluster topics around pillar pages so supporting pieces interlink (hub-and-spoke). [EXPLICIT]
- Set cadence from real capacity, not ambition (Constitution XIV Simple First). Under-commit and over-deliver. [EXPLICIT]
- Document trade-offs: depth vs. frequency, evergreen vs. timely, channel reach vs. effort. [EXPLICIT]

### Step 3: Execute
- Place each piece in a dated slot with: pillar, cluster, channel, format, owner, status, goal metric. [EXPLICIT]
- Sequence so one source asset fans out into a repurposing chain (long-form -> social -> email -> video). [EXPLICIT]
- Leave ~20% of slots open for reactive/timely content; never fill the calendar to 100%. [EXPLICIT]
- Use brand template for HTML outputs (references/brand/). Tag every claim. [EXPLICIT]

### Step 4: Validate
- Verify pillar balance, cadence realism, and that each slot has a single owner and a measurable goal. [EXPLICIT]
- Check evidence-tag coverage and Constitution compliance. [EXPLICIT]
- Confirm no orphan topics (every cluster links to a pillar) and no date collisions per channel. [EXPLICIT]

## Worked Example

Pillar "AI Adoption" | cluster "Pilot-to-Production":

- Wk 1 — 3 LinkedIn posts repurposed from pillar (goal 5% engagement)
- Wk 2 — Email digest linking the pillar (goal 25% open)

One source asset, four dated slots, one pillar. [EXPLICIT]

## Quality Criteria

- [ ] 3-5 pillars defined; every topic maps to exactly one [EXPLICIT]
- [ ] Cadence matches stated capacity; ~20% buffer left open [EXPLICIT]
- [ ] Each slot has owner, date, channel, format, and a measurable goal [EXPLICIT]
- [ ] Repurposing chains defined for source assets [EXPLICIT]
- [ ] Evidence tags on all claims; no redundancy or padding [EXPLICIT]

## Anti-Patterns

| Anti-Pattern | Why It's Bad | Do This Instead |
|-------------|-------------|-----------------|
| Random topics, no pillars | Diluted brand, unmeasurable | Map every topic to one pillar (XIII Think First) |
| Calendar filled 100% | No room for timely content; burnout | Leave ~20% buffer (XIV Simple First) |
| Over-ambitious cadence | Misses dates, erodes trust | Set cadence from real capacity |
| One asset, one post | Wastes high-effort content | Build repurposing chains |
| Slot without owner/metric | Nothing ships or gets measured | Require owner + goal per slot |
| Missing evidence tags | Claims without basis | Tag every assertion `[EXPLICIT]` |

## Edge Cases

| Scenario | Handling |
|----------|----------|
| Empty or minimal input | Request audience, channels, cadence, and goal before proceeding |
| No analytics/history | Plan from pillars; flag goals as hypotheses to validate after first cycle |
| Conflicting requirements (e.g. daily cadence, one author) | Flag the capacity conflict explicitly; propose a realistic cadence |
| Single channel only | Drop cross-channel repurposing; deepen within-channel formats |
| Calendar exceeds capacity | Cut to highest-pillar-value slots; defer the rest to next horizon |
| Timely event mid-quarter | Fill from the ~20% buffer; never displace committed pillar slots |
| Out-of-scope request | Redirect to appropriate skill or escalate |
