# Event Marketing

> "Method over hacks. Evidence over assumption."

## TL;DR

Four event sub-domains: webinar design, conference strategy, speaking proposals, event follow-up. All outputs follow MetodologIA brand standards and evidence tagging. Pick the sub-domain first, then run the 4-step procedure scoped to it. [EXPLICIT]

## Scope & Anti-Scope

- IN: agenda/flow design, audience targeting, CFP/abstract drafting, registration & nurture copy, post-event follow-up sequences. [EXPLICIT]
- OUT: paid-ad buying, venue/AV logistics, live production ops, CRM/marketing-automation config, attendee PII handling. Redirect or escalate. [EXPLICIT]
- Single-brand per output — never introduce an off-brand element. [EXPLICIT]

## Procedure

### Step 1: Discover
- Capture: objective (lead-gen | awareness | nurture | authority), audience segment, date/format, success metric, owner. [EXPLICIT]
- Read existing artifacts (past event recaps, brand templates, prior CFPs). Note current state and gaps. [EXPLICIT]
- If any of the five capture fields is missing, stop and ask — do not assume. [INFERENCIA]

### Step 2: Analyze
- Evaluate options against Constitution principles (XIII Think First, XIV Simple First). [EXPLICIT]
- Select approach with best evidence support; record the trade-off (e.g. live demo raises engagement but adds failure risk vs. recorded walkthrough). [INFERENCIA]

### Step 3: Execute
- Implement following the sub-domain checklist below; apply evidence tags to all claims. [EXPLICIT]
- Use brand template for HTML outputs (references/brand/). [EXPLICIT]

### Step 4: Validate
- Verify against Quality Criteria and the sub-domain acceptance criteria; confirm tag coverage and Constitution compliance. [EXPLICIT]

## Sub-Domain Playbooks

### Webinar design
- Structure: hook (≤2 min) → context → core value → demo/proof → CTA → Q&A. [INFERENCIA]
- Acceptance: single measurable CTA; runtime ≤45 min content + 15 Q&A; one owner per segment; fallback if live demo fails. [EXPLICIT]
- Worked example: B2B nurture webinar → CTA "book a 20-min assessment", proof = anonymized before/after, follow-up email same day. [INFERENCIA]

### Conference strategy
- Decide attend vs. speak vs. sponsor against the objective and metric from Step 1. [INFERENCIA]
- Acceptance: target-account list mapped to sessions; one primary message; pre/at/post-event touch defined per priority account. [EXPLICIT]

### Speaking proposals (CFP / abstract)
- Format: title (benefit-led) → 3-sentence abstract → 3–5 takeaways → speaker bio → why-now. [EXPLICIT]
- Acceptance: takeaways are outcomes not topics; abstract names the audience; no vendor pitch in body (organizers reject sales talks). [INFERENCIA]

### Event follow-up
- Segment by behavior: attended | registered-no-show | engaged-in-Q&A. Different message per segment. [EXPLICIT]
- Acceptance: first touch ≤24h; each segment has a distinct next step; unsubscribed/opted-out excluded. [EXPLICIT]

## Quality Criteria

- [ ] Follows Constitution principles
- [ ] Evidence tags on all claims
- [ ] Output is actionable and specific
- [ ] No redundancy or padding
- [ ] Exactly one CTA / next step per asset

## Anti-Patterns

| Anti-Pattern | Why It's Bad | Do This Instead |
|-------------|-------------|-----------------|
| Acting without understanding | Wastes effort on wrong solution | Think First (XIII) |
| Over-engineering | Complexity without value | Simple First (XIV) |
| Missing evidence tags | Claims without basis | Tag every assertion |
| Multiple CTAs in one asset | Splits attention, lowers conversion | One CTA per asset |
| Sales pitch in a CFP body | Organizers reject it | Lead with audience value |
| Generic blast follow-up | Ignores intent, burns list | Segment by behavior |

## Edge Cases

| Scenario | Handling |
|----------|----------|
| Empty or minimal input | Request the five Step-1 fields before proceeding |
| Conflicting requirements | Flag conflicts explicitly, propose resolution |
| Out-of-scope request | Redirect to appropriate skill or escalate (see Anti-Scope) |
| Multiple events in one ask | Split into one workflow run per event |
