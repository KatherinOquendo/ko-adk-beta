# Deep Variation — marketing-content

For `depth=deep`: exhaustive playbook application with per-step verification.

1. Resolve `topic`; document why the runner-up topic was rejected.
2. Load the one playbook. Confirm brand and read prior assets for continuity.
3. **Discover (full):** gather every required input; inventory proof assets
   (metrics, testimonials, citations, baselines). Flag any gap explicitly.
4. **Analyze (full):** evaluate structure/framework options against Constitution
   XIII (Think First) / XIV (Simple First); record the trade-off and why the
   runner-up structure was rejected.
5. **Execute (full):** draft to the canonical structure; map EVERY claim to a
   source and a tag; convert features to benefits; resolve or mark each
   `[SUPUESTO]` with a verification step.
6. **Validate (full):** run the complete Quality/Acceptance Criteria of the
   resolved playbook, verify each item with evidence (not assertion), and
   re-read against the topic's read-aloud / cut / clock test where defined.
7. Produce the guardian's three-gate verdict (routing, topic criteria,
   governance). Disclose any residual assumption rather than papering over it.

Output: routing-with-rationale block + fully verified artifact + itemized gate
results with evidence references.
