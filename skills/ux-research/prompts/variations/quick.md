# Quick variation — ux-research (depth=quick)

Single-pass essentials. Use when the decision is low-stakes or time is tight.

1. Resolve `topic` in one read of the request; if genuinely ambiguous, ask ONE
   question, else proceed. Set `depth=quick`.
2. Read exactly one playbook. Run its four steps at the **essentials** level:
   - `user-research` → 1–2 provisional personas + the single sharpest opportunity.
   - `survey-design` → ≤6 items, one metric (NPS/CSAT/CES) tied to the decision,
     n + MoE noted.
   - `user-testing` → 3 goal-framed tasks, success criteria, top severity-rated
     issues only.
   - `prototyping` → lowest fidelity rung that falsifies the one open assumption.
3. Output the minimal deliverable + next step. Tag claims; pair each `[SUPUESTO]`
   with a verification step. Flag anything deferred as "quick-pass gap".

Do **not** skip the decision framing or the evidence tags — those are non-optional
even in quick mode. [DOC]
