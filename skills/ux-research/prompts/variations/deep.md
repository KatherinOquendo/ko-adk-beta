# Deep variation — ux-research (depth=deep)

Exhaustive, verify-each-step. Use when the decision is high-stakes or the
deliverable feeds a build commitment.

1. Resolve `topic` + scope the decision, segments, and available primary data.
   Confirm a testable artifact/data source exists before committing the route.
2. Read exactly one playbook and run all four steps **to completion**, verifying
   each before advancing:
   - `user-research` → 2–4 research-backed personas (source per attribute), full
     empathy quadrants, end-to-end journey with sentiment curve and moments of
     truth, prioritized + owned opportunities. Reconcile qual vs quant.
   - `survey-design` → full item bank (single-construct, balanced, non-leading),
     sampling plan with min n/segment, pilot, segmented analysis with n /
     response rate / MoE, each finding tied to the decision.
   - `user-testing` → 4–6 tasks, pre-set criteria + ceilings, think-aloud,
     observation logged separately from interpretation, severity × frequency
     ratings, prioritized fixes; flag every n=1 conclusion.
   - `prototyping` → climb the fidelity ladder only as each rung passes; model
     unhappy/edge states; test with a real user/proxy before raising fidelity;
     capture what each rung falsified.
3. Pass the deliverable through the full guardian checklist. Pair every
   `[SUPUESTO]` with its verification study/step. End with an explicit next step
   and an honest status (pass/conditional/gap), never green-by-default. [DOC]
