# Marketing Context

> "Method over hacks. Evidence over assumption."

## TL;DR

Establish brand positioning, value propositions, and a messaging framework so every marketing output is consistent, on-brand, and traceable. Produces a reusable context other skills consume (pitch, copywriting, competitive-positioning), not one-off copy. All outputs follow MetodologIA brand standards and evidence tagging. [EXPLICIT]

## Inputs & Outputs

- **Inputs:** target audience/segment, the offer, competitor set, existing brand assets (tone, lexicon, `references/brand/`), the decision the messaging must drive. [EXPLICIT]
- **Outputs:** positioning statement, 1 primary + 2 supporting value props, message hierarchy, proof points (each tagged), and a do/don't lexicon. [EXPLICIT]

## Procedure

### Step 1: Discover
- Capture audience, offer, competitor set, and the action the message must trigger [EXPLICIT]
- Read existing brand assets, prior messaging, and `references/brand/`; record provenance per claim [EXPLICIT]
- Identify the single differentiator the audience cares about and can verify [EXPLICIT]

### Step 2: Analyze
- Evaluate options against Constitution principles (XIII Think First, XIV Simple First) [EXPLICIT]
- Draft positioning as: For [audience] who [need], [brand] is the [category] that [differentiator], unlike [alternative]. [EXPLICIT]
- Attach a proof point to every claim; a value prop without proof is a `[SUPUESTO]`, not a benefit [EXPLICIT]
- Document trade-offs and the rejected alternatives [EXPLICIT]

### Step 3: Execute
- Implement following established patterns; one message hierarchy, no competing primary claims [EXPLICIT]
- Apply evidence tags to all claims; cite the proof source inline [EXPLICIT]
- Use brand template for HTML outputs (`references/brand/`); single brand per artifact [EXPLICIT]

### Step 4: Validate
- Verify against quality criteria and check evidence-tag coverage [EXPLICIT]
- Confirm Constitution compliance and brand-voice consistency [EXPLICIT]
- Confirm no price claims and no green-as-success framing [EXPLICIT]

## Quality Criteria

- [ ] Follows Constitution principles [EXPLICIT]
- [ ] Evidence tags on all claims [EXPLICIT]
- [ ] Output is actionable and specific [EXPLICIT]
- [ ] No redundancy or padding [EXPLICIT]
- [ ] Exactly one primary value prop; supporting props do not compete with it [EXPLICIT]
- [ ] Every benefit claim carries a proof point + source [EXPLICIT]
- [ ] Single brand throughout; no cross-brand mixing [EXPLICIT]

## Worked Example

Audience: mid-market data teams drowning in manual reporting.
Positioning: "For data teams who lose days to manual reporting, MetodologIA is the open methodology that turns analysis into evidence-tagged outputs, unlike ad-hoc prompt hacks." [EXPLICIT]
Primary prop: cut reporting effort while keeping every claim traceable (proof: evidence-tag coverage on every output). Supporting: brand-consistent artifacts; reusable across skills. Rejected: "fastest" — unverifiable and invites a price/speed war. [EXPLICIT]

## Decisions & Trade-offs

- One primary value prop over a feature list: sharpens recall; trade-off — omits secondary benefits, so place them in the hierarchy below, not alongside [EXPLICIT]
- Proof-anchored claims over aspirational copy: builds trust and survives scrutiny; trade-off — slower to draft, blocks claims that lack evidence [EXPLICIT]
- Reusable context over per-campaign copy: keeps downstream skills consistent; trade-off — needs refresh when the offer or competitor set shifts [EXPLICIT]

## Anti-Patterns

| Anti-Pattern | Why It's Bad | Do This Instead |
|-------------|-------------|-----------------|
| Acting without understanding | Wastes effort on wrong solution | Think First (XIII) [EXPLICIT] |
| Over-engineering | Complexity without value | Simple First (XIV) [EXPLICIT] |
| Missing evidence tags | Claims without basis | Tag every assertion [EXPLICIT] |
| Multiple competing primary claims | Dilutes recall, confuses the buyer | One primary prop, rank the rest [EXPLICIT] |
| Unsubstantiated superlatives ("best", "fastest") | Invites disproof and price wars | Anchor every benefit to a proof point [EXPLICIT] |
| Mixing brands in one artifact | Breaks single-brand discipline | Identify brand first, isolate output [EXPLICIT] |

## Failure Modes

| Failure | Signal | Recovery |
|---------|--------|----------|
| No verifiable differentiator | Every claim also true of competitors | Return to Discover; narrow segment until a unique claim emerges [EXPLICIT] |
| Proof points missing | Claims tagged `[SUPUESTO]` | Demote to candidate; gather evidence before publishing [EXPLICIT] |
| Brand assets absent | No tone/lexicon in `references/brand/` | Flag `[SUPUESTO]`, propose intake, do not invent voice [EXPLICIT] |
