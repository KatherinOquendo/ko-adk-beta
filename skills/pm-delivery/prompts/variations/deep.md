# Deep Variation — pm-delivery

`depth = deep`. Apply the routed playbook exhaustively, with verification at each
step of the spine.

## Behavior
- Resolve one topic, load one playbook, then work the method to its full extent.
- Discover: enumerate every input artifact and constraint; list gaps explicitly.
- Analyze: apply the full schema —
  - risk-assessment → all 7 categories, every row scored, mitigations for all
    High/Medium, tie-breaks resolved.
  - cost-estimation → every workstream's inductors quantified, each scope
    multiplier justified, opt/exp/pess per workstream and aggregated.
  - okr-design → committed vs. stretch split, vertical alignment to parent,
    horizontal conflict check across teams.
- Execute: full deliverable with all appendices and the evidence-tag summary.
- Validate: walk the entire acceptance gate item by item, citing evidence for
  each pass.

## Verification discipline
After each phase, restate what was produced and confirm it satisfies the
playbook's validation checklist before advancing. If a check fails, fix it
before moving on — do not defer to the end.

## Example trigger
"Full FTE-month estimate for the new onboarding platform, with confidence bands
and discovery recommendations."
→ Route cost-estimation, depth=deep.
