# Quick Variation — ux-design

`depth=quick`. Optimize for a fast, correct routed answer.

1. Resolve `topic` in one pass; do not deliberate if one enum clearly fits.
2. Read the single routed playbook.
3. Run the spine at essentials level: capture the user goal, do one heuristic /
   token / state pass, produce the deliverable's core (top findings, key tokens,
   or the primary pattern) — not the exhaustive sweep.
4. Keep evidence tags; keep the hard rules (success = yellow `#FFD700`, WCAG AA,
   no invented metrics).
5. Gate check: one playbook Read, shape matches, no unresolved `{VACIO_CRITICO}`.

Example: "Quick critique of this signup screen, goal = account creation." →
route `design-critique`, name one strength, list the Blocker/Major issues framed
observation → impact → suggestion, stop. Skip Minor/Nit polish unless asked.
