# Deep Variation — brand-output

Exhaustive path for high-stakes or ambiguous branded output (`depth=deep`).

1. **Discover fully.** Enumerate content sections, brand system, language/direction,
   `artifact_date`, and external-asset permissions. Stop on `{VACIO_CRITICO}`.
2. **Route deliberately.** State the candidate topics, the disambiguating signal,
   and why the loser was rejected. If two brands fit, ask. [EXPLICIT]
3. **Execute exhaustively.** Apply every token rule; merge partial config per-key
   and record each inherited key as `[SUPUESTO]`. Generate via the deterministic
   script; for folios run `--dry-run` then `--apply`.
4. **Validate every box.** Walk the playbook's full authoritative checklist —
   semantic landmarks, `@media`, SVG favicon, WCAG AA or recorded limitation, no
   surviving placeholders, OOXML validity, tracker/document parity. Run
   `scripts/check.sh` (or the compiler). Report each box with its evidence.
5. **Audit governance.** Constitution v6.0.0, no invented prices, no PII,
   single-brand, full Alfa-tag coverage on non-obvious claims.

Use when: leave-behind for a client, pixel-correctness requested, or the route is
contested. Trade-off: more passes for auditable certainty. [INFERENCIA]
