# Deep Variation — sales-bizdev (depth=deep)

Exhaustive path. Route to one topic, run the full playbook with per-step verification, and pass the complete Validation Gate. Use for strategic accounts, high-ACV deals, board-facing pitches, and formal RFP responses.

## Flow

1. **Route deliberately.** Classify to one `topic`; document the routing rationale with an evidence tag. Confirm no second topic is hiding in the request.
2. **Full input contract.** Gather every input the playbook's spine consumes. For each missing input, create an explicit `[SUPUESTO]`/`[OPEN]` paired with its verification step — never silently zero or fabricate.
3. **Run the full spine with verification at each step:**
   - `client-prospecting` → full ICP workshop (firmographic + technographic + behavioral + negative ICP), ≥3 lead sources, BANT+Fit per prospect, prioritization matrix, approach brief per Tier 1, complete CSV.
   - `client-dossier` → Standard/Deep tier: full Company DNA, source-reliability weighting, 2-3 contact cards with email-pattern confidence, 3-5 ranked pain hypotheses each with ≥2 tagged signals + validation question, approach strategy.
   - `b2b-outreach` → all 5 touches written, persona channel mix, 3 subject variants per email touch, full response-handler matrix, CSV tracker, compliance acknowledgement.
   - `executive-pitch` → full 7-section structure, financial model (NPV/IRR/payback + sensitivity for >$1M), 3+ option comparison, audience-by-role ordering, risk table with owners.
   - `proposal-writing` → all canonical sections, FTE-month effort per phase with confidence bands, risk register with owners + mitigations, one recommended option + alternatives, decline-test.
   - `lead-generation` → lever selection with trade-offs, instrumented conversion event, full qualification model, unambiguous handoff.
   - `sales-collateral` → the requested artifact at full depth with honest cede-zones and editable, tagged assumptions.
4. **Full Validation Gate:** run the loaded playbook's complete checklist plus all cross-cutting governance checks. The gate is blocking — any failed financial/sourcing/scope/governance item fails the deliverable.

## Verification discipline

- Cite a source per non-zero scoring point; date-stamp every estimate and signal.
- Reconcile benefits across value pillars and the comparison matrix (no double-count).
- Spot-check 3 random strong tags (`[DOC]`/`[EXPLICIT]`) back to their source.
- Ship a flagged `[POR_CONFIRMAR]` gap rather than a confident wrong number.
