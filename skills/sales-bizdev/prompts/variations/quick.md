# Quick Variation — sales-bizdev (depth=quick)

Fast path. Route to one topic, run the essentials of its spine, ship a lean evidence-tagged deliverable. Use when the user needs the artifact now and exhaustive depth is not warranted.

## Flow

1. **Route in one read.** Classify to one `topic` by deliverable type. If genuinely ambiguous, ask one question — otherwise commit and load that single route.
2. **Lock brand + minimum inputs.** One brand. The minimum required input for the topic (target account, ICP one-liner, persona, or RFP). Missing → ask one targeted question, don't fabricate.
3. **Run essentials only:**
   - `client-prospecting` → ICP one-liner + top 5-10 scored prospects with tier and trigger; skip exhaustive sourcing.
   - `client-dossier` → Flash tier: Company DNA + 1 contact + 1-2 pain hypotheses + one hook.
   - `b2b-outreach` → Touch 1 + Touch 2 fully written + 3 subject variants + positive/negative response handlers.
   - `executive-pitch` → 3-section variant: problem (≥3 metrics) → solution → ask.
   - `proposal-writing` → problem reframe + scope/anti-scope + FTE-month effort + one next step.
   - `lead-generation` → one lever + named conversion event + scoring bands.
   - `sales-collateral` → one artifact answering one buying question.
4. **Gate (lean):** one tag per claim, zero currency, single brand, each `[SUPUESTO]` paired with a verify step.

## Guardrail

Quick ≠ sloppy. The non-negotiables (no prices, single brand, evidence tags, no fabricated proof, no PII) hold at every depth. If quick mode would force a fabricated number or a trigger-less Tier 1, stop and flag the gap instead.
