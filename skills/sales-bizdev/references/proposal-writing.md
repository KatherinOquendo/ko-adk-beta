# Proposal Writing
> "Method over hacks."
## TL;DR
Turn an RFP/discovery into a win-shaped proposal: problem reframe, scope + anti-scope, FTE-month effort, phased timeline, risk register, pricing structure (NOT prices). [EXPLICIT]
## Procedure
### Step 1: Discover
- Extract buyer's stated need, unstated pain, decision criteria, and evaluation rubric from RFP/transcript [EXPLICIT]
- Identify decision-maker vs. champion vs. blocker; tailor framing to the economic buyer [INFERENCIA]
### Step 2: Analyze
- Evaluate options per Constitution XIII/XIV; pick ONE recommended path, name 1-2 alternatives with trade-offs [EXPLICIT]
- Size effort in FTE-months per phase with a confidence band; never quote a price or rate [EXPLICIT]
### Step 3: Execute
- Draft sections in this order: Problem → Outcome → Scope/Anti-scope → Approach → Timeline → Team → Risks → Commercial structure → Next step [EXPLICIT]
- Tag every claim about the buyer's environment with its source tag [EXPLICIT]
### Step 4: Validate
- Verify quality criteria + acceptance criteria below; do a "decline test": would a skeptical CFO sign? [EXPLICIT]

## Proposal Structure (canonical section order)
| Section | Must answer | Anti-pattern |
|---------|-------------|--------------|
| Problem | What does the buyer lose by doing nothing? | Restating the RFP verbatim |
| Outcome | Measurable end-state, not activities | "We will help you transform" |
| Scope / Anti-scope | What is and is NOT included | Open-ended scope (margin killer) |
| Timeline | Phases, gates, dependencies, who-owns-what | Single waterfall bar with no gates |
| Risks | Top 3-5, each with owner + mitigation | Generic "schedule risk" boilerplate |
| Commercial | Structure (T&M / fixed / phased), FTE-months | Prices, day rates, discounts [EXPLICIT] |

## Quality Criteria
- [ ] Evidence tags applied to every environmental claim
- [ ] Constitution-compliant (no prices, single-brand, no green-as-success)
- [ ] Actionable output (clear next step + owner + date)

## Acceptance Criteria
- [ ] Anti-scope section present and non-empty [EXPLICIT]
- [ ] Effort expressed in FTE-months with confidence band, zero currency figures [EXPLICIT]
- [ ] Each risk has a named owner and a concrete mitigation [EXPLICIT]
- [ ] Exactly one recommended option; alternatives carry trade-offs [EXPLICIT]
- [ ] Single brand throughout — no off-brand elements [EXPLICIT]

## Worked Example (condensed)
> Problem: "Manual invoice matching costs ~3 FTE and delays close by 5 days [SUPUESTO — confirm in discovery]."
> Outcome: "Close cycle from 10→5 days; matching automated for 80% of volume."
> Scope: OCR + 3-way match. Anti-scope: ERP migration, master-data cleanup.
> Effort: Phase 1 discovery 0.5 FTE-mo; Phase 2 build 2-3 FTE-mo (±30%).
> Commercial: phased, gate after Phase 1. (No price stated.) [EXPLICIT]

## Assumptions & Limits
- Assumes access to RFP/discovery artifacts (transcripts, docs, configs) [EXPLICIT]
- Requires English-language output unless otherwise specified [EXPLICIT]
- Does not replace domain expert judgment or legal/commercial sign-off [EXPLICIT]
- Produces structure + effort sizing, NOT a priced quote [EXPLICIT]

## Edge Cases & Failure Modes
| Scenario | Handling |
|----------|----------|
| Empty or minimal input | Request clarification before proceeding |
| Conflicting requirements | Flag conflicts explicitly, propose resolution |
| Out-of-scope request | Redirect to appropriate skill or escalate |
| Buyer asks for a price | Provide FTE-month structure + disclaimer; route pricing to human [EXPLICIT] |
| No decision criteria given | State assumed criteria as [SUPUESTO]; confirm before submission [EXPLICIT] |
| Multiple brands plausible | Stop, identify single brand FIRST, then draft [EXPLICIT] |
| Scope keeps expanding | Lock anti-scope; move extras to a named Phase 2 [INFERENCIA] |
