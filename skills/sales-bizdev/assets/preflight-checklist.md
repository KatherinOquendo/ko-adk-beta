# Pre-flight Checklist — sales-bizdev

Run before declaring any sales-bizdev deliverable done. Maps to the guardian gate and the per-playbook Validation Gates.

## Routing

- [ ] Request classified to exactly one `topic` (∈ enum); rationale recorded with a tag.
- [ ] Only one route file loaded; the cluster was not bulk-loaded.
- [ ] Channel words were stripped before classifying (LinkedIn/email/deck/call ≠ topic).
- [ ] Genuine tie or no fit → one disambiguating question was asked, not a double load.

## Context

- [ ] Single brand identified before drafting; no brand mixing.
- [ ] `depth` resolved (quick = essentials, deep = exhaustive + per-step verification).
- [ ] Required inputs present; each missing input flagged `[SUPUESTO]`/`[OPEN]` with a verify step (not fabricated).

## Governance (every output)

- [ ] Every non-obvious claim carries one tag; one family, one spelling.
- [ ] Zero currency figures — FTE-months + disclaimer only.
- [ ] No client PII beyond public professional context.
- [ ] No fabricated proof point, metric, case study, or contact detail.
- [ ] No green-as-success styling; confidence stated, not implied.

## Topic-specific gate (apply the loaded one)

- [ ] client-prospecting → BANT + Fit per prospect; every Tier 1 has a dated trigger + approach brief.
- [ ] lead-generation → named conversion event + unambiguous fit×intent handoff.
- [ ] client-dossier → right entity verified; 3-5 pain hypotheses (≥2 signals each + validation Q); ends in one hook.
- [ ] b2b-outreach → ICP score first; fresh (<90d) trigger in Touch 1/2; full 5-touch + 3 subject variants + response handlers + CSV.
- [ ] executive-pitch → problem ≥3 metrics; financial model; 3+ options; named decision owner; no fabricated budget.
- [ ] proposal-writing → non-empty anti-scope; FTE-month effort + confidence band; each risk owned + mitigated; one recommended option.
- [ ] sales-collateral → one buying question; honest cede-zone; brand template.

## Close

- [ ] Deliverable ends in a concrete next action with an owner.
- [ ] Every `[SUPUESTO]`/`[OPEN]` paired with its verification step.
