# Variation — Quick (seo-growth)

`depth = quick`. Deliver the essentials of one playbook fast, without skipping Validate.

## When to use
The user wants a focused, high-signal answer for one topic — a meta block, a single ICE
ranking, a rendering recommendation, a top-line indexability score — not an exhaustive pass.

## Tight loop
1. Resolve the one topic (ask only if genuinely ambiguous).
2. Read that one playbook.
3. **Discover** only the inputs the deliverable strictly needs.
4. **Analyze + Execute** the single highest-value output:
   - `seo-architecture` → rendering call + the one canonical/sitemap fix.
   - `seo-content` → the meta block + one `<h1>`/heading check.
   - `indexability-validator` → the score line + Missing/Orphan counts.
   - `conversion-optimization` → the top ICE item + one hypothesis.
   - `funnel-design` → the TOFU/MOFU/BOFU stage map skeleton.
   - `social-proof` → the single best-fit trust signal for the stated risk.
5. **Validate** against the playbook's top acceptance criteria — still mandatory.

## Constraints
Evidence-tag claims; mark any missing baseline as a gap; no invented numbers; single brand.
If `quick` cannot honestly satisfy the request (e.g. a full funnel blueprint), say so and
recommend `deep`.
