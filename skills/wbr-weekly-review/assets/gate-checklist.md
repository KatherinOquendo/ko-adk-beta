# Gate checklist — WBR Weekly Review

Checklist que el guardian recorre antes de declarar un acta hecha. Todos deben
marcarse para pasar. {DOC}

- [ ] Encabezado con periodo ISO + alcance + fecha de corte.
- [ ] Las 6 secciones presentes (avances, estancado, friccion, cumplimiento previo,
      compromisos, arrastres).
- [ ] Cada estancado con dueno **y** proximo paso fechado (sin "se revisara").
- [ ] Cada compromiso previo marcado cumplido / parcial / no-cumplido; fallidos no
      omitidos.
- [ ] Cada friccion clasificada recurrente vs puntual; recurrentes (≥2 semanas)
      escalan a accion.
- [ ] ≤5 compromisos para la proxima semana, cada uno con criterio de hecho.
- [ ] Arrastres ≥3 semanas marcados `{POR_CONFIRMAR}` y escalados a bloqueo.
- [ ] Cada avance con evidencia ligada (`{DOC}`/`{ADJUNTO}`); nada verde sin evidencia.
- [ ] Una sola familia de tags Jarvis `{...}`; ningun `[...]` en el documento.
- [ ] Si todo verde y nada estancado → se pregunto que NO se movio (anti-subreporte).
