# Gate checklist — daily-close

El guardian no declara "hecho" hasta que cada casilla este marcada. Refleja el
gate de aceptacion de `SKILL.md` y la rubrica de `assets/quality-rubric.json`.

## Ejes
- [ ] Cerrado, Pendiente y Aprendido presentes.
- [ ] Ningun eje vacio sin justificacion explicita ("sin avances" cuenta).

## Evidencia
- [ ] Cada item Cerrado liga artefacto verificable (commit / archivo / ancla).
- [ ] Cada Cerrado lleva `{DOC}` o `{EXTRAIDO_HILO}` con traza.

## Semilla
- [ ] 1-3 pendientes priorizados (no la lista entera).
- [ ] Cada semilla tiene su primer paso ejecutable en frio.

## Tags
- [ ] Una afirmacion no obvia = exactamente un tag Jarvis `{...}`.
- [ ] Sin tags Alfa `[...]` en el documento (familia unica).
- [ ] Ningun `{WEB}` sin cita.

## Persistencia
- [ ] Cierre aplicado como append; historico intacto.
- [ ] Bloque fechado al dia real (no al de captura si es retroactivo).
- [ ] Un solo workspace (marca unica); sin fusionar bitacoras.
- [ ] Bloqueos que cruzan al manana marcados `{POR_CONFIRMAR}` + paso.

Si alguna casilla falla → veredicto distinto de `pass`; devolver al lead con el
check nombrado. No entregar cierre parcial.
