# Quick variation — data-governance (depth=quick)

Fast path. Resolve the topic, load one playbook, apply the essentials only.

1. **Resolve topic** in one pass using the weakest-overlap rule. State the chosen
   topic and a one-line reason. Ask a question ONLY on a genuine tie.
2. **Load** the single `references/<topic>.md`.
3. **Apply essentials**:
   - privacy → classify PII, pick one de-identification technique, name lawful basis.
   - audit-trail → give the minimum record schema + hash-chain note.
   - documentation → dictionary columns + lineage source per field.
   - governance/strategy → roles + one escalation path + top decision.
   - pipeline → the four gates and the ≥0.95 confidence bar.
   - storytelling → one insight, framed for the audience.
4. **Validate lite**: one playbook loaded, output on-topic, evidence tags present,
   no prices/PII/brand-mix.

Keep it tight — the most decision-relevant 20%. No exhaustive enumeration.
